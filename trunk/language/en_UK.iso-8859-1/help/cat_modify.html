Edit a category

Informatons

  • name: rename the category (a virtual rename in case of physical category.
  • description: give a general idea of the content, this text will be display under the thumbnail page.

Move

If the category is virtual, you can move it. Moving a category means changing its parent category.

Options

  • Access type: Permission management. If you make a category private, all its child categories becomes private. If you make a category public, all its parent categories becomes public.
  • Lock: The category and its sub-categories will temporary been disabled for maintenance.
  • Comments: Authorize users to comment elements of this category.
  • Authorize upload: Authorize users to upload files. This option is available only if category is physical (not virtual).

You can also manage these options with "Properties" management screen (screens Upload, Comments, Lock, Public/Private, Representative available from Administration, Categories, Properties).

Sort order

Choose if the default sort order will be used.

Indicate if this sort order will be applied to subcategories.

Select specific sort by columns to use with this category.

  • Creation date: Picture creation date
  • Post date: Synchronization date
  • Average rate (*): Average rate could be changed by the visitor
  • Most visited (*): The current visit change the most visited order
  • File name: Given name in informations field
  • Id: Internal Id (recent categories have an higher id than previous ones.

(*)Take care and test it your self can give unpredictable results.

Representant

The representant of the category is the thumbnail displayed to represent the category on main page (category.php) when a category contains only sub-categories (just as the root category).

There are 4 ways to set the representative picture of a category :

  • picture.php: in the action buttons bar, one of them let you set the displayed picture as representative for the displayed category. This button is available only for administrators.
  • Modify informations about a picture in administration. This screen can be reach from picture.php or Batch management in unit mode. See the help of this screen for details.
  • Administration, Categories, Properties, Representative. See the help of this screen for details.
  • Edit a category (this screen).

The representation depends on configuration parameter allow_random_representative (see include/config_default.inc.php).

In default mode (allow_random_representative set to false) each category containing at least one element is represented by a fixed element. Once set (at category creation), the representative changes only when an administrator ask for it. If the representant is not appropriate, you can ask to Find a new representant by random.

If configuration parameter allow_random_representative is set to true, a category containing elements can have no fixed representant. Just use Delete representant button.

If the category contains no element (but only sub-categories) it can be represented by any element thanks to screen Modify informations about a picture. The only option on the current screen is to Delete representant button, after using this button, no representant option to set anymore.

Link all category elements to a new category

  • Virtual category name: New category name to be created, all pictures of the current category will be linked to the new one.
  • Parent category: Where the new category will be, leave empty to create it from home.
